- 2026-06-23 - Hotfix de resposta async apos audio/anexo: mensagens com midia eram enfileiradas e o `claudeWorker` era executado, mas a chamada worker -> `ai-gateway` retornava 401 com `Wrong recipient, payload audience != requiredAudience`; o frontend ficava apenas com a bolha "Anexo enviado." sem resposta da IA. Correcao: o worker passou a enviar o ID token tambem em `X-Serverless-Authorization`, mantendo `Authorization` para a validacao OIDC app-level do gateway, separando a autenticacao IAM do Cloud Run da autenticacao interna do Express. Testes: suite focada de worker/media/payload 14/14, `npm run lint`, `npm run build` e suite completa 174/174 verdes.
- 2026-06-23 - Hardening pos-auditoria de seguranca: `api` e `claudeWorker` passaram a declarar service accounts dedicadas no deploy (`pt-api-runtime` e `pt-worker`, com override por env), `api` ficou explicitamente publica por IAM e `claudeWorker` ficou restrita a `pt-tasks-invoker` via `invoker`, mantendo a validacao OIDC no codigo. `BOOTSTRAP_SECRET` foi vinculado via Secret Manager apenas na `api`. CSP saiu do modo report-only por padrao, o Hosting passou a enviar `Content-Security-Policy` enforcement sem `script-src 'unsafe-inline'`, o registro do service worker foi movido para `public/sw-register.js`, e CORS passou a bloquear `*.ngrok-free.app` em producao mesmo se cair na allowlist. A fila `pt-ai-responses` foi reduzida para `maxDispatchesPerSecond=2` e `maxConcurrentDispatches=3`; o bucket `mw-personal-trainer.firebasestorage.app` passou para UBL habilitado e public access prevention enforced, preservando lifecycle de `pt-media/`. Dependencias foram atualizadas para remover vulnerabilidades de producao e dev (`npm audit --omit=dev` e audit completo zerados localmente apos overrides controlados de `uuid` e `brace-expansion`).
- 2026-06-23 - Gateway e controle de custo em producao: `gateway` (Cloud Run) foi alterado para `ingress=all` mantendo IAM restrito a service accounts (`run.invoker` sem `allUsers`/`allAuthenticatedUsers`) para permitir que `api` e `claudeWorker` alcancem a URL direta `https://gateway-ewylrqiwjq-rj.a.run.app`. `api` e `claudeWorker` foram redeployados com `AI_GATEWAY_URL`/`AI_GATEWAY_AUTH_AUDIENCE` apontando para a URL Cloud Run, `TOKEN_BUDGET_ENABLED=true`, caps de token (`TOKEN_BUDGET_DAILY=500000`, `TOKEN_BUDGET_MONTHLY=5000000`) e caps de custo estimado por usuario (`TOKEN_BUDGET_DAILY_COST_MICROS=500000`, `TOKEN_BUDGET_MONTHLY_COST_MICROS=5000000`, equivalentes a US$0,50/dia e US$5/mes).
